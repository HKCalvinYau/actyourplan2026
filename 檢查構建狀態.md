# 🔍 檢查構建狀態和問題排查

## 請先確認以下事項

### 1. 是否已修改構建命令？

在 Cloudflare Pages 設定中，Build command 是否已改為：
- ✅ `npm run pages:build`
- ❌ 還是 `npx @cloudflare/next-on-pages@1`？

### 2. 構建是否正在進行？

1. 登入 Cloudflare Dashboard
2. 進入您的專案 `actyourplan2026`
3. 點擊 **Deployments** 標籤
4. 查看最新的部署狀態：
   - 🟡 **Building** = 正在構建中（請等待）
   - 🟢 **Success** = 構建成功
   - 🔴 **Failed** = 構建失敗（需要查看錯誤）

### 3. 如果構建失敗，查看錯誤日誌

1. 點擊失敗的部署
2. 點擊 **View build log** 或 **Build log**
3. 查看錯誤訊息

**常見錯誤：**

#### 錯誤 A: `npm run pages:build` 找不到
```
npm ERR! missing script: pages:build
```
**原因：** 代碼還沒更新到最新版本
**解決：** 確認 GitHub 上的代碼已更新，然後重新部署

#### 錯誤 B: `@cloudflare/next-on-pages` 找不到
```
Error: Cannot find module '@cloudflare/next-on-pages'
```
**原因：** 需要安裝適配器
**解決：** 我來幫您添加依賴

#### 錯誤 C: 構建超時
```
Build timeout
```
**原因：** 構建時間過長
**解決：** 可能需要增加構建時間限制

## 🔧 快速檢查步驟

### 步驟 1: 確認 GitHub 代碼已更新

訪問：https://github.com/HKCalvinYau/actyourplan2026

確認最新的 commit 是：
- `Fix: Add pages:build script for Cloudflare Pages adapter`

如果沒有看到這個 commit，可能需要等待幾分鐘讓 GitHub 同步。

### 步驟 2: 確認 Cloudflare 構建命令

在 Cloudflare Pages 設定中：
- **Build command** 應該是：`npm run pages:build`
- 不是：`npx @cloudflare/next-on-pages@1`

### 步驟 3: 手動觸發重新部署

1. 在 Cloudflare Pages 專案中
2. 點擊 **Deployments**
3. 找到最新的部署
4. 點擊 **Retry deployment** 或 **Redeploy**

### 步驟 4: 等待構建完成

構建通常需要 5-10 分鐘，請耐心等待。
